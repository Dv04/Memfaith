\begin{abstract}
As Large Language Models (LLMs) scale to millions of tokens using memory agents, the gap between semantic retrieval and causal utilization becomes a critical concern. This report introduces MemFaith, a framework for measuring causal faithfulness using Leave-One-Chunk-Out (LOCO) ablation. We define the Causal Chunk Score (CCS) and evaluate it across single-hop (FEVER) and multi-hop (HotpotQA) tasks. Our results demonstrate that causal influence dilutes as memory pressure increases and reveals distributed causal necessity in multi-hop reasoning.
\end{abstract}

\section{Introduction}
The development of memory-augmented LLMs allows for processing vast contexts. However, it remains unclear whether these models causally rely on the stored memory or fall back on parametric priors. MemFaith pivots from expensive parametric edits like ROME to a scalable inference-time ablation strategy to quantify this reliance.

\section{Methodology}
\subsection{Causal Chunk Score (CCS)}
For a context segmented into $K$ chunks, the CCS is defined as:
\[
\mathrm{CCS}(K) = \frac{1}{K} \sum_{i=1}^K \mathbb{I}(A_{-i} \neq A_{\text{full}})
\]
where $A_{\text{full}}$ is the baseline answer and $A_{-i}$ is the answer generated after ablating chunk $i$.

\subsection{Pipeline Architecture}
The MemFaith pipeline consists of:
\begin{enumerate}
    \item \textbf{Deterministic Chunker}: Sentence-aware segmentation using spaCy.
    \item \textbf{Long-Context Constructor}: Interleaving evidence with distractors to simulate memory pressure.
    \item \textbf{Inference Engine}: Deterministic greedy decoding ($\text{temp}=0.0$).
    \item \textbf{Causal Comparator}: Regex matching for FEVER and Token-F1/LLM-as-a-judge for HotpotQA.
\end{enumerate}

MemFaith provides a rigorous metric for evaluating how memory agents utilize long-context information. Our ablation-based approach enables both mechanistic understanding and practical context pruning through behavioral distillation.
