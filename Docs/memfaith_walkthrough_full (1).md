## Scope principle used while writing

The docs resolve the ambiguity in the current proposal by treating:
- **CCS / chunk ablation** as the mainline project,
- **learned discriminator / pruning** as an extension,
- and **EF / ROME integration** as a bonus or future-work direction.

That choice was made because the project window only runs through the week of April 20, so the planning documents must reflect a schedule that can actually finish.

## Main difference from the previous too-short version

The earlier compressed version was too small and too summary-like.

The new pack was written to be:
- implementation-first,
- schedule-aware,
- team-aware,
- and much closer in spirit to the longer ML-with-graphs planning document and work-division artifacts.
