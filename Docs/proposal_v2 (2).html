MemFaith

Measuring Causal Faithfulness of Memory Agents in Long-Context LLMs
4 Graduate Students Knowledge Editing Long-Context Agents Mechanistic Interpretability
01

Motivation & Research Question

Memory agents such as MemAgent (Yu et al., 2025) process arbitrarily long documents by reading in segments and maintaining a compressed memory buffer via an overwrite strategy, achieving <5% performance loss on 3.5M-token QA. However, a fundamental question remains unanswered: does the memory the agent writes actually causally drive its downstream answers?

Our teammate's XAI seminar work introduced the Editability–Faithfulness (EF) metric, which uses knowledge editing (ROME) to test whether perturbing a rationale-cited fact flips the model's answer more than a matched control edit. On FEVER + GPT2-XL, EF ≈ 12.5% overall and up to 22.9% on refutes claims — indicating partial but noisy causal grounding.

The core tension this project investigates is:

"As overwrite depth K increases, does the memory agent's causal control over rationale-cited facts (EF score) monotonically degrade? And when external memory conflicts with internal parametric knowledge, which source does the model preferentially trust?"

This extends EF from single-turn rationale evaluation to multi-segment memory agents, directly combining both lines of prior work into a novel evaluation framework.

02

Experimental Setup & Definitions

2.1 — Core Variable Definitions

Memory Pressure
Defined as Total Document Tokens / Max Buffer Size. Controls how aggressively the overwrite strategy must compress information.
Overwrite Steps K
Primary experimental variable. K ∈ {0, 2, 4, 8}. K = 0 is the direct-context baseline (no memory overwrite).
Edit Type: ROME
Parametric edit — modifies model weights to change internal factual belief. Tests whether the model follows its parameters even when memory text says otherwise.
Edit Type: Text Edit
Context edit — directly modifies the string in buffer M_K. Baseline comparison to isolate whether parametric vs. contextual grounding dominates.

2.2 — Data & Model

Dataset
FEVER dev set (SUPPORTS / REFUTES / NEI labels) with appended Wikipedia distractor passages to increase document length and memory pressure.
Base Model
GPT2-XL (1.5B) — same as teammate's baseline, enabling direct comparison. Optional: Qwen2.5-1.5B as a more capable alternative.
Memory Agent
Simplified MemAgent: sliding-window reader with configurable buffer. M ← overwrite(M, segment_i) at each step. K controls total overwrite depth.
03

Detailed Methodology

Stage 1
Inference. Model reads final memory M_K, generates answer A_pre and free-text rationale R.
Extraction. Triple extractor identifies target factual triple T_tgt = (s, r, o) from rationale R.
Control Construction. Build T_ctrl with same relation r but substitute object o′ not appearing in the original document.
Stage 2
ROME Intervention. Apply parametric edit to model weights: change internal belief about T_tgt to a counterfactual object. Memory text M_K remains unchanged.
Control Edit. Apply identical ROME edit to T_ctrl to isolate edit collateral noise.
Conflict Test. Re-run QA with edited model on same M_K. The model must choose: follow the (now-wrong) parametric memory, or trust the (still-correct) context buffer?
Stage 3
Compute EF(K). For each overwrite depth K, compute the EF score across N examples (3 seeds × 30).
Degradation Curve. Plot EF vs. K. Hypothesis: EF degrades as K increases, as overwrite dilutes the causal link between any source fact and M_K.
Stratification. Report EF by gold label (SUPPORTS / REFUTES / NEI) and by memory length tertile, following teammate's protocol.
★ Novel Design: The Conflict Test

By applying ROME to model weights while keeping M_K text intact, we create a deliberate parameter–context conflict. The flip rate after this conflict reveals whether the agent's memory acts as genuine external grounding or merely echoes the model's parametric prior — a question not previously studied in the memory agent literature.

Target Edit (ROME)
Edit weights for T_tgt. M_K unchanged. → Re-run QA → A_tgt. Measures: does parametric overwrite override the agent's memory?
Control Edit (ROME)
Edit weights for T_ctrl. M_K unchanged. → Re-run QA → A_ctrl. Measures: baseline noise from non-causal parametric perturbation.
EF(K) = Pr[Atgt ≠ Apre] − Pr[Actrl ≠ Apre]
Faithfulness as a function of overwrite depth K — the core experimental variable
04

Evaluation Criteria

Metric Description Purpose
EF Score Difference in flip rates between target and control edits, computed per K and stratified by gold label / memory length. Primary measure of causal faithfulness
Conflict Rate Proportion of examples where parameters were edited but model still followed the text buffer M_K (i.e., answer unchanged). New metric introduced in this work. Distinguish parametric vs. context dependence
EF vs. K Curve Plot EF across K ∈ {0, 2, 4, 8}. Test whether faithfulness degrades monotonically with overwrite depth. Core hypothesis validation
QA Accuracy Exact match / F1 on FEVER dev set before any edits. Ensures agent baseline is functional. Sanity check
Bootstrap 95% CI Confidence intervals on all EF values, 3 seeds × 30 samples each (= 90 total per K), following teammate's protocol. Statistical validity
Flip-Type Distribution Categorize each example: target-only flip (✓), control-only flip (✗), both flip, neither flip. Qualitative case studies per type. Interpretability of EF signal
05

Limitations & Future Work

⚠ Limitations
Model scale: ROME is practically constrained to ~1.5B models; results may not generalize to instruction-tuned or larger LLMs.
Extraction noise: Heuristic triple extractor (constant score = 1.0) introduces spurious control-only flips — the primary EF noise source identified in the preliminary work.
Memory pressure ceiling: Current K values may not reach the collapse threshold; higher compression ratios remain unexplored.
Simplified agent: Sliding-window approximation may not reproduce the RL-trained overwrite dynamics of the original MemAgent.
Single domain: FEVER covers short, single-hop facts; multi-hop faithfulness under memory overwrite is untested.
→ Future Work
Scale to Qwen2.5-7B + MEMIT for multi-token editing on larger, instruction-following models.
Replace heuristic extractor with LLM-assisted triple extraction to reduce EF noise and improve targeting precision.
Extend to HotpotQA / MuSiQue for multi-hop faithfulness evaluation under memory overwrite.
Combine with a context compressor (e.g., LLMLingua) in the segment reader; compare compression-induced vs. overwrite-induced EF degradation.
Explore memory repair: if EF degrades at high K, can targeted re-reading of key segments restore causal grounding?
