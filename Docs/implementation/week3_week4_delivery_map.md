# MemFaith Week 3-4 Delivery Map

This document outlines the Phase 3 and Phase 4 deliverables completed for the MemFaith CCS project.

## Dev Sanghvi (Technical Lead)

### Phase 3: Multi-Hop Adaptation & Handoff
- **Enhanced Answer Comparison**: Integrated `MockLLMJudge` for semantic equivalence checks in HotpotQA tasks when Token-F1 is ambiguous ($F1 < 0.6$).
- **Multi-Hop Dependency Analysis**: Implemented `src/memfaith/multi_hop_analysis.py` to quantify distributed causal necessity in multi-hop reasoning.
- **Stratified Analysis**: Added `src/memfaith/stratification.py` for slicing CCS by gold label, chunk position, and dataset.
- **Combined Label Export**: Created `scripts/export_combined_labels.py` to merge FEVER and HotpotQA logs for the discriminator training track.
- **Qualitative Review**: Developed `scripts/review_hotpot_flips.py` to extract and document causal flips for manual inspection.

### Phase 4: Synthesis & Hypothesis Testing
- **Publication-Ready Visualization**: Implemented `src/memfaith/plotting.py` generating:
  - CCS Degradation Curves
  - Label-based CCS Stratification
  - Positional Flip-Rate Bar Charts
  - Gold vs Non-Gold Causal Comparison
  - Multi-Hop Dependency Heatmaps
- **Case Study Extraction**: Automated 6-category case study extraction in `scripts/extract_case_studies.py`.
- **Reproducibility**:
  - `run_eval.sh`: One-command execution of the entire evaluation pipeline.
  - `Makefile`: Integrated new evaluation and plotting targets.
  - `results/memfaith_report.tex`: New LaTeX report structure for MemFaith CCS findings.

## Cross-Team Integration
- **Synthetic Scaling**: Created `scripts/generate_synthetic_data.py` to generate 100+ example datasets for pipeline verification.
- **Unified Interface**: Updated `src/memfaith/__init__.py` to expose all new analysis and plotting utilities.
- **Validation**: Verified the full pipeline end-to-end using the synthetic data suites.

---

*Note: Final empirical results on real datasets (Wikipedia/HotpotQA full) will be generated by running the `run_eval.sh` script with real-data backends.*
